Componentes del proyecto

🖥️
Virtualización con Linux ▼
1 · Instalación de las máquinas virtuales

Arranque del instalador de Kubuntu 24.04 LTS en VirtualBox — VM gráfica del equipo.

Escritorio de Kubuntu ya instalado y funcionando correctamente.

2 · Particionamiento manual

Kubuntu: particiones Sistema EFI, Swap, Root (/) y Home creadas manualmente.

Ubuntu Server: particiones /, /boot/efi, /home y SWAP configuradas manualmente.

3 · Configuración de red entre las VMs

Kubuntu: interfaz enp0s8 en la red interna con IP 192.168.10.2.

Ubuntu Server: interfaz enp0s8 con IP 192.168.10.3 — ambas VMs en la misma red interna.

4 · Acceso SSH funcional

Conexión SSH exitosa desde Kubuntu hacia grupo4@192.168.10.3 (Ubuntu Server).

Servicio ssh.service activo y corriendo en Ubuntu Server, con sesiones registradas.

Contenedores Docker ▼
Instalación de Docker Engine

Docker Engine y Docker Compose instalados correctamente en Kubuntu 24.04 LTS (Docker v29.1.3 · Compose v5.1.4).

Prueba con docker run hello-world: el motor de Docker descarga, ejecuta y reporta el contenedor de prueba correctamente.

Reflexiones

Lo que aprendimos

01
Virtualización como base
Configurar VMs desde cero nos permitió entender el ciclo completo de un sistema operativo: particionamiento, red y comunicación entre máquinas.
02
Contenedores y portabilidad
Docker mostró cómo aislar servicios y sus dependencias para lograr despliegues reproducibles sin importar el entorno subyacente.
03
Orquestación real
Minikube nos acercó a la gestión de contenedores a escala: réplicas, recuperación automática y exposición de servicios — pilares del cloud moderno.
04
Trabajo en equipo
Git permitió coordinar el trabajo del grupo, mantener trazabilidad de cambios y publicar evidencias de forma accesible para todos.
05
Visión integral
Unir virtualización, contenedores y orquestación en un solo proyecto nos dio perspectiva real de cómo funciona la infraestructura cloud de extremo a extremo.
06
Proyección profesional
Linux, Docker y Kubernetes son estándar en la industria — este proyecto es un primer paso sólido hacia roles en DevOps y cloud engineering.
