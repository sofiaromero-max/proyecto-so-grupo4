Componentes del proyecto

🖥️
Virtualización con Linux ▼
1 · Particionamiento en Kubuntu

Particionamiento manual: Sistema EFI (512 MB), raíz / (9.77 GB ext4), Swap (2 GB) y /home (7.73 GB ext4).

2 · Configuración de red en Kubuntu

Verificación de interfaz de red con ip a en Kubuntu. IP asignada en la red interna de VirtualBox.

3 · Particionamiento en Ubuntu Server

Almacenamiento en Ubuntu Server: particiones /, /boot/efi, /home y SWAP creadas manualmente.

4 · Red e IP en Ubuntu Server

Verificación con ip a en Ubuntu Server. IP 192.168.10.3 — comunicación entre VMs establecida.

5 · Conexión SSH exitosa

Acceso SSH desde Kubuntu al Ubuntu Server (192.168.10.3) verificado exitosamente. Sistema Ubuntu 26.04 LTS corriendo correctamente.

Evidencias: particionamiento · configuración de red · comunicación entre VMs
🐳
Contenedores Docker ▼
  • Docker Engine instalado en Kubuntu 24.04 LTS
  • Dockerfile personalizado — Frontend: Nginx sirviendo HTML estático
  • Dockerfile personalizado — Backend: servidor Python en puerto 5000
  • Orquestación completa con docker-compose.yml
  • Volúmenes persistentes configurados para los archivos HTML
# Dockerfile — Frontend (ubuntu:24.04) FROM ubuntu:24.04 RUN apt-get update && apt-get install -y nginx COPY index.html /var/www/html/ EXPOSE 80 CMD ["nginx", "-g", "daemon off;"] # Levantar ambos servicios docker compose up -d
Evidencias: Dockerfiles · docker-compose.yml · frontend puerto 80 · backend puerto 5000
⚙️
Orquestación con Kubernetes ▼
  • Minikube instalado en Kubuntu 24.04 LTS
  • Manifiestos YAML: Deployment y Service
  • Nginx oficial desplegado con 2 réplicas iniciales
  • Servicio expuesto con tipo NodePort para acceso externo
  • Escalado manual demostrado hasta 3 réplicas
kubectl apply -f deployment.yaml kubectl get pods kubectl get svc # Escalar a 3 réplicas kubectl scale deployment nginx-deployment --replicas=3 minikube service nginx-service
Evidencias: kubectl get pods · kubectl get svc · acceso navegador · escalado

Reflexiones

Lo que aprendimos

01
Virtualización como base
Configurar VMs desde cero nos permitió entender el ciclo completo de un sistema operativo: particionamiento, red y comunicación entre máquinas.
02
Contenedores y portabilidad
Docker mostró cómo aislar servicios y sus dependencias para lograr despliegues reproducibles sin importar el entorno subyacente.
03
Orquestación real
Minikube nos acercó a la gestión de contenedores a escala: réplicas, recuperación automática y exposición de servicios — pilares del cloud moderno.
04
Trabajo en equipo
Git permitió coordinar el trabajo del grupo, mantener trazabilidad de cambios y publicar evidencias de forma accesible para todos.
05
Visión integral
Unir virtualización, contenedores y orquestación en un solo proyecto nos dio perspectiva real de cómo funciona la infraestructura cloud de extremo a extremo.
06
Proyección profesional
Linux, Docker y Kubernetes son estándar en la industria — este proyecto es un primer paso sólido hacia roles en DevOps y cloud engineering.
