// 02 — implementación

Componentes del Proyecto

C1 Virtualización con Linux 30% ▼
  • Instalación de 2 VMs en VirtualBox: Kubuntu 24.04 LTS (gráfica) y Ubuntu 24.04 Server (consola)
  • Particionamiento manual con 3 particiones: /, swap, /home en ambas VMs
  • Red interna configurada entre las dos VMs para comunicación directa
  • SSH activado y verificado — acceso desde la VM gráfica hacia la VM consola
# Verificar SSH desde Kubuntu → Ubuntu Server ssh usuario@192.168.56.10 # Ver particiones en la VM de consola lsblk df -h
📸 Evidencias: capturas de instalación · particionamiento · configuración de red · prueba SSH funcional
C2 Contenedores Docker 30% ▼
  • Instalación de Docker Engine (no Docker Desktop) en Kubuntu 24.04 LTS
  • Dockerfile personalizado para Frontend: Nginx sirviendo HTML estático — basado en ubuntu:24.04
  • Dockerfile personalizado para Backend: servidor HTTP Python en puerto 5000
  • Orquestación con docker-compose.yml — ambos servicios levantados juntos
  • Volúmenes persistentes configurados para los archivos HTML del frontend
# Dockerfile — Frontend (Nginx) FROM ubuntu:24.04 RUN apt-get update && apt-get install -y nginx COPY index.html /var/www/html/ EXPOSE 80 CMD ["nginx", "-g", "daemon off;"] # Levantar servicios docker compose up -d docker compose ps
📸 Evidencias: Dockerfiles · docker-compose.yml · capturas de navegador accediendo al frontend (puerto 80) y backend (puerto 5000)
C3 Orquestación con Kubernetes 20% ▼
  • Instalación de Minikube en Kubuntu 24.04 LTS
  • Manifiestos YAML creados: Deployment y Service
  • Imagen oficial de Nginx desplegada con 2 réplicas iniciales
  • Servicio expuesto con tipo NodePort para acceso externo desde el navegador
  • Escalado manual demostrado: de 2 a 3 réplicas
# Desplegar y verificar kubectl apply -f deployment.yaml kubectl get pods kubectl get svc # Escalar a 3 réplicas kubectl scale deployment nginx-deployment --replicas=3 # Acceder al servicio minikube service nginx-service
📸 Evidencias: kubectl get pods · kubectl get svc · acceso desde navegador · demostración de escalado
C4 Sitio Web de Documentación 10% ▼
  • Sitio publicado en GitHub Pages — accesible públicamente
  • Secciones cubiertas: Home, Equipo, Componentes, Conclusiones
  • Diagramas de arquitectura, capturas de evidencias y comandos principales incluidos
  • Video demostrativo en YouTube (máximo 10 minutos)

// 03 — reflexiones

Conclusiones

🖥️
Virtualización como base
La instalación y configuración de VMs con Linux nos permitió comprender el ciclo completo de despliegue de sistemas operativos, desde el particionamiento manual hasta la comunicación en red entre máquinas.
🐳
Contenedores y portabilidad
Docker demostró cómo los contenedores aíslan aplicaciones y sus dependencias, facilitando el despliegue reproducible de servicios sin importar el entorno subyacente.
⚙️
Orquestación con Kubernetes
Minikube nos introdujo a la orquestación real de contenedores: gestión de réplicas, auto-recuperación y exposición de servicios — fundamentos de cualquier sistema cloud moderno.
🤝
Trabajo colaborativo
El uso de Git y GitHub Pages como plataforma central permitió coordinar el trabajo del equipo, mantener historial de cambios y publicar evidencias técnicas de forma accesible.
📐
Arquitectura integrada
Integrar virtualización, contenedores y orquestación en un único proyecto nos dio visión de cómo se construyen infraestructuras cloud reales, desde el servidor físico hasta el servicio expuesto.
🚀
Proyección profesional
Las tecnologías abordadas — Linux, Docker y Kubernetes — son estándar en la industria tecnológica, por lo que este proyecto representa un primer paso sólido hacia roles DevOps y Cloud.
